User Login Tests

Test Case 1: Login with Valid Credentials
Test Case ID: UL-01
Description: Verify that the user can log in using valid email and password.
Precondition: User is already registered in the system.

Steps:
1. Open the login page.
2. Enter a valid email address.
3. Enter the correct password.
4. Click the Login button.

Expected Result:
- User is successfully logged in.
- User is redirected to the dashboard.


Test Case 2: Login with Invalid Password
Test Case ID: UL-02
Description: Verify system behavior when an incorrect password is entered.
Precondition: User is already registered.

Steps:
1. Open the login page.
2. Enter a valid email address.
3. Enter an incorrect password.
4. Click the Login button.

Expected Result:
- Login fails.
- Error message is displayed indicating invalid credentials.


Test Case 3: Login with Unregistered Email
Test Case ID: UL-03
Description: Verify login attempt using an email that is not registered.
Precondition: Email does not exist in the system.

Steps:
1. Open the login page.
2. Enter an unregistered email address.
3. Enter any password.
4. Click the Login button.

Expected Result:
- Login fails.
- Error message is displayed.


Test Case 4: Login with Empty Fields
Test Case ID: UL-04
Description: Verify validation when login fields are empty.
Precondition: None.

Steps:
1. Open the login page.
2. Leave email and password fields empty.
3. Click the Login button.

Expected Result:
- Login is not allowed.
- Validation messages appear for required fields.

Test Case 7: Login with SQL Injection Attempt
Test Case ID: UL-07
Description: Verify system security against SQL injection.
Precondition: None.

Steps:
1. Open the login page.
2. Enter ' OR '1'='1 in the email field.
3. Enter any password.
4. Click the Login button.

Expected Result:
- Login fails.
- System remains secure and no data is exposed.
