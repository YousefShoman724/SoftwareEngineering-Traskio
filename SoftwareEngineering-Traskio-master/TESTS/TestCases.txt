AUTHENTICATION TEST CASES
=====================================

Test Case ID: TC-01
Title: Signup with Valid Data
Description: Verify that a new user can successfully sign up with valid email and password.
Precondition: User is not registered.

Steps:
1. Open the signup page.
2. Enter a valid email address.
3. Enter a valid password.
4. Enter matching confirm password.
5. Click the Signup button.

Expected Result:
- Account is created successfully.
- User is redirected to login page.


Test Case ID: TC-02
Title: Signup with Existing Email
Description: Verify system prevents duplicate email registration.
Precondition: Email already exists.

Steps:
1. Open the signup page.
2. Enter an already registered email.
3. Enter a password.
4. Confirm the password.
5. Click the Signup button.

Expected Result:
- Signup fails.
- Error message is displayed.


Test Case ID: TC-03
Title: Signup with Password Mismatch
Description: Verify validation when password and confirm password do not match.
Precondition: None.

Steps:
1. Open the signup page.
2. Enter a valid email.
3. Enter a password.
4. Enter a different confirm password.
5. Click the Signup button.

Expected Result:
- Signup fails.
- Error message indicates password mismatch.
